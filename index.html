Chapter 6: Events

Event Handling Process:

  1. Select Element Nodes you want a script to respond to
  2. Indicate which event will trigger a response(called "binding")
  3. Code the response to your liking

Three ways to bind an event to an element

  • HTML Event Handlers

    DO NOT USE! It is bad practice(cuz it's better to put all code in a JS file, NOT inside HTML) you may see this in older code

  • Traditional DOM Event Handlers

    Better than using html approach and is supported by most major browsers, the disadvantage is that it only allows attaching ONE function to an event - this might cause conflict

    Example: element.onevent = functionName; JS line 2


  • DOM Level 2 Event Listeners

    Most preferred way of binding BUT may not be supported by much older browsers, allows one event to triggeer multiple functions - much less likely to cause conflict

    Example:element.addEventListener('event', functionName [,Boolean]);

    Takes three parameters:

    i: event you wanna listen for

    ii: code you want it to run

    iii: Boolean indicating how events flow

    Example at JS Line 16


Using Parameters W/ Event Handlers & Listeners

Because you can't add parenthesis() after function names during the binding process, there is a way to work around it

To work around this, wrap the function call with an anonymous function

Example: element.addEventListener('blur', function() { checkInputLength(5) };, false);

USER INTERFACE EVENTS

User Interface(UI): Events that occur from interaction with the browser window rather than HTML contained within it(IE: page loaded, window resize)

  • load: triggered when page finishes loading, can also trigger nodes of elements that load images/scripts/objects
  • unload: triggered when web page is unloading(usually when a new page is requested)
  • error: triggered when there is a JS error or asset that tdont exist
  • resize: triggered when window is being resized
  • scroll: triggered when user scrolls up/down a screen, can be used for a specific element/entire page

Focus & Blur Events

focus/focusin:triggered when elements gains focus

blur/focusout:triggered when element loses focus


MOUSE EVENTS

  • click:triggered when user clicks primary mouse button, or when enter is pressed on keyboard when an element is in focus. also works with touchscreen single tap
  • dblclick: double click/tap, pretty self explanatory
  • mousedown: when user clicks any mouse botton(touchtart for touch function)
  • mouseup: when mouse button is released(touchend for touch function)
  • mouseover: when cursor is moved inside a element
  • mouseout: when cursor is moved out of the current element and to another one
  • mousemove: when mouse cursor is being moved inside an element(repeated)

    • mouseup & mousedown are separated for drag & drop features/game development functionality
